# Rocket Item

## Goal

Add a comically large, purchasable rocket item. The player can pick it up, place it, and then interact with it to trigger a 5-second launch sequence — camera locks onto the rocket as it ascends, the player is hidden, and the game transitions to a new Game Over scene near the end of the flight.

## What changes

### 1. Rocket item (`game/items/rocket.lua`)

- Inherits from `Item` base class.
- Sprite: `assets/images/items/rocket.png` at **120×240 px** (5× the 48px knife in height, comically tall and narrow).
- `_type = "rocket"`, `carriable = true` so F-key pickup/drop works normally.
- `interact(player, scene, scene_manager)` — called when player is empty-handed and presses E nearby:
  1. Set `self._launched = true`, `self._flight_timer = 0`.
  2. Drop from player hands: `self.held = false`, `player.held_item = nil`.
  3. Signal the scene: `scene.active_rocket = self` and `scene.rocket_scene_manager = scene_manager`.
- `update(dt)` — when `_launched`:
  - Move upward: `self.y = self.y - 300 * dt` (300 px/s, fast enough to leave screen in ~5s).
  - Increment `_flight_timer`.
  - At `_flight_timer >= 4.5` seconds: call `scene.rocket_scene_manager:switch(GameOverScene.new(scene.game_state))`.
  - Stop moving after transition is triggered (avoids double-trigger).

### 2. Game Scene changes (`game/scenes/game_scene.lua`)

- New state fields: `self.active_rocket = nil`.
- In `update(dt)`:
  - **Camera**: if `self.active_rocket` is set, follow the rocket centre instead of the player. No bound-clamping — the camera should track the rocket off the green world area into darkness.
  - **Player**: if `self.active_rocket` is set, skip `self.player:update(dt, self)` (player is frozen).
- In `draw()`:
  - If `self.active_rocket` is set, exclude the player from the Y-sorted entity list (player is hidden).
  - The rocket itself is already in `self.items` and will draw normally via Y-sort.

### 3. Game Over scene (`game/scenes/game_over_scene.lua`)

- `GameOverScene.new(game_state)` — receives the live `game_state` to read final money and animals sold.
- Draws a full-screen overlay:
  - Large "GAME OVER" heading.
  - Money earned and animals sold summary lines.
  - "Press R to restart" prompt.
- On R press: switches to a fresh `GameScene.new(scene_manager)`.
- Uses the CRT shader (consistent with ShopScene).

### 4. Shop catalogue (`game/scenes/shop_scene.lua`)

- Add Rocket entry to `CATALOGUE`:
  ```lua
  { name = "Rocket", cost = 300, desc = "A one-way ticket skyward.", constructor = Rocket.new, image = <rocket img> }
  ```

### 5. Detector (`game/systems/detector.lua`)

- Add `Detector.is_rocket(e)` — returns `e._type == "rocket"`.
- Update `can_pickup()` to include `"rocket"` as a valid pickup type.

### 6. Placeholder PNG (`assets/images/items/rocket.png`)

- Generated programmatically at 120×240 px using LÖVE's `ImageData` API in a one-off script, or drawn as a solid red rectangle with a pointed top as a stub — to be swapped later with real art.
- Because LÖVE cannot write PNGs at runtime in the normal game loop, the placeholder will be **generated by a small standalone Lua script** run once via `love .` with a special entry point, or simply committed as a hand-crafted minimal PNG.

---

## What stays the same

- All existing items (Wire Roll, Knife, Breeder) are untouched.
- The F-key pickup / E-key interact split is unchanged — rocket follows the same pattern.
- Shop UI layout and navigation are unchanged; the new item just appears as a 4th catalogue entry.
- Animal breeding, job system, sell bin, and all other game logic are untouched.
- Scene transition uses the existing `scene_manager:switch()` fade mechanic.

## Open questions

None — all design decisions resolved before writing this doc:

| Question | Decision |
|----------|----------|
| Launch trigger | Place rocket, walk up, press E (interact while empty-handed) |
| Player during flight | Hidden (excluded from draw, input frozen) |
| Game Over content | Money earned + animals sold summary, press R to restart |
| Rocket sprite | Placeholder PNG at 120×240 committed to repo |
| Rocket cost | $300 |
| Flight speed | 300 px/s upward, scene transition triggered at t=4.5s |
